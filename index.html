모든 대학생을 위한
LMS(Learning Management System)


2017112674 박현준 / 2017112690 김무영

선정 배경

장애 학생의 대학 진학률이 높아지는 상황에서 COVID-19가 급속도로 확산되었다.
대학들은 비대면 수업 방식으로 학기를 진행했고, 장애 대학생들은 새로운 도전 상황을 맞게 되었다.
본 프로젝트 팀은 이러한 어려움들을 접근성이 낮은 LMS의 문제로 집약하고 이를 개선해보기로 하였다.

기존 LMS 분석 및 개선 방안

이클래스 분석


마우스 없이 키보드만으로 사용 가능한 웹 디자인(지체장애/상지장애)
동국대학교 이클래스 ‘내 강의실’의 ‘#aside’ 바와 동국대학교 이클래스 첫 페이지의
사이버캠퍼스, 커뮤니티, MOOC 메뉴 Navigation bar의 드롭다운(하위 메뉴)은 tab으로 접근이 불가능함
→ tab키로 접근 가능한 웹 페이지 요소 구성 및 디자인
과도한 텍스트의 팝업 이미지화
공지사항과 같은 중요한 정보는 글뿐만 아니라 이미지로 제공할 필요가 있음
→ 공지사항 별로 내용을 잘 설명할 수 있는 팝업 이미지를 만들어 대체
자막 제공(청각장애)
동국대학교 LMS에서는 실시간/비실시간 자막을 기본적으로 제공하지 않고 있음
사운드, 오디오 등을 청취할 수 없는 환경에서 강의 영상을 볼 수 없으므로, 개선이 필요
→ 네이버 클로바, 구글 스피치 API를 사용하여 모든 학생들에게 자막 서비스를 제공
음성 해설(시각장애/전맹)
동국대학교 LMS의 일부 요소는 alt 속성이 적용되어 있지 않음
모니터를 볼 수 없는 환경에서 모니터 내의 시각적 정보를 청각적으로 접근 가능하도록 모든 중요한 Element에는 alt 속성이 필요
→ 중요한 웹 페이지 요소를 선별하여 alt 속성을 부여
글자 크기 확대, 돋보기, 색상 반전 기능 제공(시각장애/저시력, 색맹)
동국대학교 LMS에서는 해당 기능을 제공하지 않고 있음
해당 기능을 별도의 프로그램, 장비 없이 웹 사이트 자체적으로 제공하도록 개선할 필요가 있음(ex. 국립특수교육원, 널리)
→ 해당 기능을 이용할 수 있는 버튼을 웹 페이지에 추가

결과



  • 기존 이클래스 웹페이지를 토대로 HTML, CSS, Javascript 기반 제작
  • LMS 개선 제안서를 작성하여 정보처에 제출

    Q. ATM처럼 특정 장애를 위한 별도의 웹 페이지를 제공하는 것이 좋을까요? 아니면 지금처럼 유니버셜 디자인을 하는 것이 좋을까요?
    Q. 클로바, 구글 스피치 API는 아직 정확도가 다소 떨어지고, 속기사는 많은 인건비를 필요로 합니다. 합의점을 어떻게 보는 것이 좋을까요?
    Q. 저희가 놓친 통합교육의 시선에서 본 이클래스 개선점이 있다면 알려주세요!
